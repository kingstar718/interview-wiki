# 高频题目索引

> 大厂高频题,按热度排序。分三部分:
> - **A. 高频算法题**:刷题优先级,从上往下刷。
> - **B. 高频八股题**:背诵优先级,自上而下过。
> - **C. 各公司风格与高频方向**:针对性准备。
>
> 热度 ★★★★★ 几乎必考 / ★★★★ 高频 / ★★★ 常见。
> 公司标注为大致倾向,以实际为准:[阿里][腾讯][字节][美团][百度][京东][拼多多][滴滴][网易][快手]。

## C. 各公司风格与高频方向

> 风格为社区总结的大致倾向,具体以岗位/部门为准。

| 公司 | 风格 | 高频方向 |
|---|---|---|
| 阿里 | 重项目深度、分布式/中间件深挖、系统设计 | JVM、MySQL、分布式事务、秒杀、RocketMQ |
| 字节 | 重算法(偏难,手撕多)、扣细节 | 困难 DP/图/单调栈、Kafka、Redis、epoll |
| 腾讯 | 计网/操作系统基础扎实、协议细节 | TCP、HTTPS、进程线程、C++/Go |
| 美团 | 八股全、业务场景多 | MySQL、Redis、Spring、本地生活场景题 |
| 百度 | 基础 + 算法 | 数据结构、网络、排序 |
| 京东 | Java 八股、并发 | 并发、JVM、中间件 |
| 拼多多 | Java 核心、加班考察 | 并发、JVM、MySQL |
| 滴滴 | 中间件、业务场景 | MQ、Redis、出行场景 |
| 网易 | Java 基础、方向相关 | 集合、并发、游戏/云音乐方向 |
| 快手 | 算法 + Redis | 中等偏难算法、缓存设计 |

## 复习建议

1. **算法**:A 表 1~25 必刷且能默写;26~40 至少有思路。每题写 Java 解 + 复杂度 + 一句变式。
2. **八股**:B 表 1~30 必背,能口述 2~3 分钟;31~50 理解原理。
3. **公司**:按目标公司查 C 表,针对性补短板。
4. **节奏**:算法每天 2~3 题保持手感;八股按领域过,每领域总结一篇 md。
