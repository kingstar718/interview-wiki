# 279. 完全平方数（Perfect Squares）

频次 ★★★ · 难度 🟡 · 高频：字节

## 题目

给定正整数 n，找到若干个完全平方数（如 1, 4, 9, 16...）使其和为 n，返回最少的个数。

**示例**：
```
输入: n = 12
输出: 3  （4+4+4）

输入: n = 13
输出: 2  （4+9）
```

## 思路

**完全背包（DP）**：`dp[i]` 表示凑出 i 所需的最少完全平方数个数。

每个完全平方数 `j*j` 视作一个物品（无限使用），转移：
`dp[i] = min(dp[i], dp[i - j*j] + 1) for j*j ≤ i`

初始化 `dp[0] = 0`，其余为 INF（n + 1）。

也可以用 **BFS（四平方和定理剪枝）**，但 DP 写法更通用。

## 代码

```java
public int numSquares(int n) {
    int[] dp = new int[n + 1];
    Arrays.fill(dp, n + 1);
    dp[0] = 0;
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j * j <= i; j++) {
            dp[i] = Math.min(dp[i], dp[i - j * j] + 1);
        }
    }
    return dp[n];
}
```

## 复杂度

- **时间**：O(n√n) —— 外层 n，内层最多 √n
- **空间**：O(n)

## 边界条件

- n = 0：不存在（题目 n 正整数，但 DP 基态 dp[0] = 0）
- n = 1：dp[1] = 1（1²）
- n 本身是完全平方数：内层首次 j² = n 时 dp[n] = 1

## 变式

- **[322. 零钱兑换](322-coin-change.md)**：同样是完全背包求最少数，物品改成了硬币
- **四平方和定理**：每个自然数都可表示为 4 个整数的平方和，BFS 解法可基于此剪枝，但最值 DP 更通用

## 易错点

- **INF 初始化**：n 最多用 n 个 1² 构成，所以 `n + 1` 作为 INF 足够
- **写法对称性**：外层 for i 内层 for j 和反过来都行（求最值，顺序无关），但如果是求组合数注意外物品内容量

## 面试追问

- **四平方和定理能不能优化？** 可以，先用"一个平方数"判，再"两个平方数"判（双指针），都失败则答案为 3 或 4，最坏 O(√n)。但 DP 的 O(n√n) 对 n ≤ 10⁴ 已经够快，面试写 DP 最稳妥
- **和 322 关系？** 本质完全一样：物品集合不同而已，322 是硬币面额，本题是 1², 2², 3²...
